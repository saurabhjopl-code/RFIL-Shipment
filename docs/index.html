Shipment Packing Workflow
Rajnandini Fashion India Ltd

1. Purpose

This document explains the end-to-end business logic for a packing workflow involving two roles — Creator and Packer — used in Amazon & Flipkart shipment preparation.

2. Roles and Permissions

Creator

  • Create shipment drafts using temp shipment ID format: Location + DDMMYYYY.
  • Edit drafts until packing is submitted.
  • Confirm submitted shipments after verifying data.
  • Export Excel reports.

Packer

  • View pending shipments.
  • Open shipment → status becomes Working.
  • Scan FNSKU barcodes into auto-created boxes.
  • Close boxes; create new boxes.
  • Submit shipment when packing is complete.

3. Shipment Status Workflow

  • Draft → Created by Creator.
  • Working → Packer opens shipment.
  • Submitted → Packer submits; Creator must confirm.
  • Confirmed → Final export is generated + email sent.

4. Packing Logic

Box Numbering

Boxes auto-create in sequence: BLR5-1, BLR5-2, BLR5-3...

Barcode Scan Logic

  • Each scan = qty 1.
  • Scan added as line item: Box#, FNSKU, Qty.
  • Live grid shows scanned items.
  • Packer may close a box → system auto-opens next box.

On Submit → Packer can no longer edit.

5. Creator Confirmation

Creator must enter shipment temp ID (e.g., BLR511122025) to confirm.

System then:

  • Generates Packing ID (e.g., RAJ-BLR5-20251111-001).
  • Creates consolidated Excel export.
  • Emails final file to operations team.

7. Edge Cases

  • Over-scan (scanned qty > expected) → block submit.
  • Offline scanning → local autosave.
  • Wrong FNSKU → flag for manual review.
  • Creator can edit only before confirmation.
